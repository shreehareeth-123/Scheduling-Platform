Tech Stack

Built on a battle-tested modern stack

Every technology choice was intentional — optimizing for developer velocity, production reliability, and long-term scalability.

Frontend
React 18 + Vite
TailwindCSS 3.4 for utility-first styling, React Query for server-state management, FullCalendar v6 for calendar UI components.
v18.3 · Vite 5.2
Backend
Node.js + Express 4
RESTful API with JWT authentication, bcrypt password hashing, helmet for security headers, express-rate-limit for DDoS protection.
Node v20 LTS · Express 4.18
Database
PostgreSQL 16 + Prisma
Relational schema for users, events, availability slots, and bookings. Prisma ORM with migrations and type-safe queries.
Prisma 5.x · pg 16
Calendar API
Google Calendar v3
Full OAuth2 PKCE flow using googleapis v140. Reads existing events for conflict detection, writes new events on booking confirmation.
googleapis v140 · OAuth2
Time Zones
date-fns-tz + Luxon
All datetimes stored as UTC in DB. Frontend detects viewer's timezone via Intl.DateTimeFormat API and displays slots in local time.
date-fns 3.x · Luxon 3.x
Deployment
Vercel + Railway
Frontend on Vercel CDN with automatic preview deployments. Backend on Railway with PostgreSQL add-on. CI/CD via GitHub Actions.
GitHub Actions · HTTPS enforced

How the booking flow works

1
Host sets availability
Host connects Google Calendar via OAuth2, then sets weekly recurring available hours in the dashboard.
2
Guest visits booking link
Guest opens the shareable link — no account needed. Available slots are fetched in real-time, excluding existing Google Calendar events.
3
Select date & time
The UI auto-detects the guest's timezone and shows times in their local time. Guest picks a slot and enters their name + email.
✓
Booking confirmed
Event created in Google Calendar for both parties. Confirmation email with .ics attachment sent instantly via Nodemailer.
Submission Summary

Key decisions & assumptions

Key Decision
Google Calendar as Source of Truth
Rather than building custom event storage and syncing, Google Calendar is the authoritative store for the host's schedule. This eliminates double-booking race conditions entirely.
Assumption
Single-Host Scope for MVP
Multi-tenant architecture (teams, organizations) is designed in the DB schema but not deployed in this 7-day sprint. The schema supports it without migrations.
Tech Choice
React + Node.js for Velocity
Chose a shared-language (JS/TS) fullstack to maximize speed. The team can contribute across frontend and backend without context-switching.
LLM Usage
Claude Used Throughout
Used Claude (Anthropic) for boilerplate generation (Prisma schema, Express routes), code review passes, writing unit test stubs, and producing this presentation and website.
Security
Defense in Depth
JWT RS256 signing, bcrypt cost-factor 12, HTTPS enforced, helmet CSP headers, express-rate-limit on auth endpoints, parameterized queries via Prisma (no SQL injection).
Scalability
Designed to Scale
Stateless API allows horizontal pod scaling. Prisma connection pooling, Redis-ready session store interface, CDN-deployed frontend, and DB indexes on booking lookup paths.
